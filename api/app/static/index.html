For more details and source code, see github.com/bil/timestamping

Trusted Timestamping FAQ

This is a website for generating trusted timestamps. Trusted timestamps are useful for establishing that a specific piece of data existed in that exact form at a given time (and has not been altered since). Among its applications, this can be useful for scientific data integrity.

The data to be timestamped is processed locally on the web browser, and a hash digest is calculated. This digest, which contains no details or information about the originating data, is sent to a server in the cloud through a Web API that performs the timestamping against a number of public timestamp authority servers in compliance with RFC 3161.

The timestamp generated by the server is then:

  • made available to the user for download
  • stored on the server
  • made public at github.com/bil/timestamp-record

There are a number of uses of trusted timestamping. Most scientific-related uses involve asserting data integrity. This can be experimental scientific data or sections of an electronic lab notebook.

A researcher may want to prove that a given piece of data has not changed since it was collected. This could be useful to assert data integrity during publication, even if years later. It can also be used to prove conception/invention dates of new scientific ideas/methods/tools/etc.

There are no data privacy conerns with timestamping or this service. Data never leaves your local computer. The hash digest is calculated in the browser window via the JavaScript Crypto library using SHA-256. SHA-256 is a one-way operation that yields a unique number (one of 2^256 possibilities) generated by the original input data. Importantly, the algorithm cannot be run in reverse to generate the original data, and it is effectively impossible to find different piece of data that yields the same digest.

For these reasons, there is no data privacy risk in sharing the digest, or even making it public (as this service does). The digest is the only item sent to the server, and there is no additional information actively gathered. The digest does not identify the data or the user in any way, unless the data owner voluntarily reveals the data themselves.

IP addresses and connection times are captured as part of default cloud logging services--a VPN can mask the originating IP if desired.

There is no obligation to use this web service to generate timestamps. Timestamps can be generated locally without the web service using the openssl wrapper scripts available at github.com/bil/timestamping. The scripts at this repository are used by this web service to generate its timestamps. One additional benefit of using the web service is that it acts as an Evidence Recording Authority which can provide long-term guarantees of otherwise limited lifetime timestamps (explained further in a different question).
